# ritual.md

> You opened this file. Good. That's already the first step.

Did you know? `testing.md` exist! When a feature is ~80% complete (use your best judgement or ask the agent) and it's' working,
then that means the time has cometh, the moment is at hand. Take a deep breath, be careful.


### Before You Start

The project serves you. You don't serve the project.

Now, read these in order:
1. `README.md` — what this project currently is. Out of date? Update!
2. What does the git shortlog say? Last three to five entries should be enough.
3. `TODO.md` — if it exists, pick a bullet point off what's next, bite-sized


### Before You Stop

 > For the love of what's good, remove expired TODO entries, alright?

Ask your agent to write down something you plan on doing next, append TODO.md.

That's it. One or two next actions, concrete enough that future-you can start it
in under thirty seconds without having to think about what it means.

**If this session revealed new patterns** (debugging approaches, doc structures, workflow improvements):
- Extract what worked into a temp file
- Take a short break
- Propose changes to spec-template, iterate a bit, integrate

The goal is to jump-start your future self, be it a new project or a new session.


### Security & Sanity Pass

Keep the spec, the readme and code in-sync. Do security and sanity checks.

What could go wrong? Who could abuse this? What's the attack surface?

- Auth / access control considered?
- Any irreversible operations (deletes, sends, charges)? Guarded?
- External inputs validated and untrusted?
- Secrets out of source control?
- Dependencies vetted (not abandoned, not obviously malicious)?
- Anything that could surprise a solo maintainer six months from now?


### When You're Stuck

You are not stuck because you're lazy, but likely because the next action is too large or too vague.

Break the task in half. Then break that half in half. When it's small enough to feel almost embarrassingly easy -- do that.
Ask others, find who was doing the exact same thing -- how they do it?

When was the last time you had something to eat? Are you hydrated? Is the oxygen alright? Please, find some time to sleep!
